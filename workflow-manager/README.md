# Workflow Manager

Reusable workflow documentation toolkit for any repository.

## What is included
- `docs/_system/`: canonical templates and examples (`epic`, `feature`, `task`, `poc`)
- `docs/scripts/`: automation scripts for GitHub sync and PR creation
- `docs/prompts/`: reusable prompts for documentation and workflow audits
- `docs/injector/`: optional sample-data injector scaffold
- `docs/WORKFLOW_INDEX.md`: mobile-friendly reading map and glossary

## Project-specific spaces (intentionally empty by default)
- `docs/features/`
- `docs/planning/`
- `docs/runbooks/`
- `docs/guides/`
- `docs/workflow/`
- `docs/tasks/`
- `docs/epics/`

Each consuming project should populate those folders with its own markdowns.
