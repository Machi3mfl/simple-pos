# Prompt Library

This folder stores reusable prompts for significant feature deliveries and documentation maintenance.

## Why this folder exists
- Keep high-value prompts versioned in the repository.
- Reuse proven workflows for future feature cycles.
- Preserve prompt quality and consistency across contributors.

## Naming convention
Use stable IDs and short slugs:

- `PRM-001-docs-refresh-and-workflow-audit.md`
- `PRM-002-feature-delivery-closeout.md`
- `PRM-003-release-readiness-audit.md`

## Suggested prompt structure
Each prompt file should include:

1. Objective
2. Context
3. Required steps
4. Constraints
5. Expected output format
6. Verification checklist
